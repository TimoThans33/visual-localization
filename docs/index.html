Visual Localization Design Document

This document describes the proposed software architectures for indoor localization of autonomous machines.

Introduction

This project aims to localize autonomous robots using image processing algorithms. I have heard we sometimes do image processing at Prime Vision ;p It has to at least include the following features:

  • Optic Flow for speed estimation of the Robot
  • Absolute Positioning
  • No Need for seperate devices (except for maybe an accesspoint and server) or adjustments on the environment
  • Less then 5mm precission
  • More than 30 updates per second
  • Ability to learn while driving
  • Algorithms rather implemented from scratch
  • Cheaper than 2000$

Pre-liminary work

We have build three set-ups for testing. First is a PCB design for the Ilumination. The second is a cover for the PCB design and camera. The third is a cart that can be either pushed or driven with a controller. The camera in the set-up is a basler camera. The images appear to be of a good enough quality to do something. But this design is not final.

Furtheremore there is some throw-away code for real-time video streaming to see the feed on your computer

Proposed solutions

Optic Flow Algorithm

Can both be used with a floor and camera pointed to the ceiling.

Image Stitching Algorithm with SIFT

Maybe in combination with WLAN based Localization. Could be useful with both a floor and ceiling camera

K Means Clustering

Maybe in combination with WLAN based localization and could also be used with both a floor and cealing camera

Relative Positioning with depth data

True swarming algorithm. Determine its position within the fleet of robots.
